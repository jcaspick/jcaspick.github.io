Sugarcube

August 7, 2019

Sugarcube is a generative art program that creates intricate voxel structures using 3D cellular automata. The program is written in C++ and uses GLFW for creating the window and OpenGL context. The GUI is built using ImGui, an excellent open source project that I can't say enough nice things about. This is the first non-trivial program I wrote while learning OpenGL.

You can find the the source code on github, feel free to modify and expand on it if you are so interested. You can also download a pre-built binary here (windows executable) if you'd like to try making some images without compiling the program from source. It is very much a toy program and the interface is a little rough but I've tried to include lots of helpful tooltips.

The program has two simple shaders, one of which colors the structure using a distance based ramp/gradient, and another which colors the faces based on their direction. These images can be brought into photoshop or some other graphics program and combined to create a variety of different effects.

How it works

Anybody who shares my interest in generative art has likely heard of cellular automata before. They come up a lot in creative coding circles, probably because they are fun to tinker with and give complex and aesthetically pleasing results while being relatively simple to implement even for beginner programmers. The most famous cellular automata is “Conway's Game of Life,” in which whole ecosystems of growth and activity seem to emerge from inert clumps of pixels.

My simplified and not-at-all-mathematically-rigorous explanation of cellular automata is as follows:

  • They are patterns of cells in a regular grid structure (most often a 2D square grid, but in my case a 3D volume of voxels)
  • The patterns evolve over time in discrete steps
  • The next step of the system is computed by applying simple rules to each cell of the current step.

The state of the pattern at each step is described by whether each cell is “alive” (filled in) or “dead” (empty). The rules for Conway's Game of Life for example are as follows:

  • A live cell with fewer than two live neighbors dies
  • A live cell with two or three live neighbors is unchanged
  • A live cell with more than three live neighbors dies
  • A dead cell with exactly three live neighbors becomes a live cell

If you try using Sugarcube you'll notice that the rules of the system can be set by changing four cryptic variables: eL, eU, fL, and fU. These four numbers are a simple notation for describing Conway's game of life and all conceptually similar cellular automata (known as “life-likes”). They represent the lower and upper bounds of two ranges (hence L and U). Restated in terms of these values, the rules of a life-like cellular automata are as follows:

  • a live cell must have at least eL neighbors and at most eU neighbors to continue living
  • a dead cell must have at least fL neighbors and at most fU neighbors to become a live cell

Note that not all patterns will result in any sort of growth and it's quite easy to come up with impossible rules that cause the starting shape to die after a single generation.
